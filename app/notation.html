According to the WCA,

  • 12a) Notation for 3x3x3 puzzles:
    • 12a1) Face Moves:
      • 12a1a) Clockwise, 90 degrees: F (front face), B (back face), R (right face), L (left face), U (upper face), D (bottom face).
      • 12a1b) Counter-clockwise, 90 degrees: F', B', R', L', U', D'.
      • 12a1c) 180 degrees: F2, B2, R2, L2, U2, D2.
    • 12a2) Outer Block Moves (outer slice plus adjacent inner slices):
      • 12a2a) Clockwise, 90 degrees: Fw, Bw, Rw, Lw, Uw, Dw.
      • 12a2b) Counter-clockwise, 90 degrees: Fw', Bw', Rw', Lw', Uw', Dw'.
      • 12a2c) 180 degrees: Fw2, Bw2, Rw2, Lw2, Uw2, Dw2.
  • 12b) Rotations for 3x3x3 puzzles:
    • 12b1) Clockwise, 90 degrees: z, z', x, x', y, or y'.
    • 12b2) Counter-clockwise, 90 degrees: z', z, x', x, y', or y.
    • 12b3) 180 degrees: z2, x2, or y2.

According to Superset ENG, it is convinient to use middle layer moves. These are also commonly abbreviated S, S', M', M, E', and E respectively.

  • s1) Middle Layer Moves (slice move plus rotation):
    • s1a) Clockwise, 90 degrees: Fm, Bm, Rm, Lm, Um, Dm.
    • s1b) Counter-clockwise, 90 degrees: Fm', Bm', Rm', Lm', Um', Dm'.
    • s1c) 180 degrees: Fm2, Bm2, Rm2, Lm2, Um2, Dm2.

According to Jessica Fridrich, it is also convinient to consider slice moves and antislice moves.

  • f1) Slice Moves (For example, Rs = Ls' = R L', and Ls = Rs' = R' L)
    • f1a) Clockwise, 90 degrees: Fs, Bs, Rs, Ls, Us, Ds.
    • f1b) Counter-clockwise, 90 degrees: Fs', Bs', Rs', Ls', Us', Ds'.
    • f1c) 180 degrees: Fs2, Bs2, Rs2, Ls2, Us2, Ds2.
  • f2) Antislice Moves (For example, Ra = La = R L, and La' = Ra' = R' L')
    • f2a) Clockwise, 90 degrees: Fa, Ba, Ra, La, Ua, Da.
    • f2b) Counter-clockwise, 90 degrees: Fa', Ba', Ra', La', Ua', Da'.
    • f2c) 180 degrees: Fa2, Ba2, Ra2, La2, Ua2, Da2.

These notations do not cover all possible parallel moves. To enumerate all parallell moves for compression purposes, we need a table.

R\L 0 1 2 3
0 N/A L L2 L'
1 R L R = Ra L2 R = Lr L' R = Rs
2 R2 L R2 = Rl L2 R2 = Rs2 = Ra2 L' R2 = Rl'
3 R' L R' = Rs' L2 R' = Lr' L' R' = Ra'

Glossary

The state of the cube at any given time is called a position. There are usually an infinite number of algorithms (Face and Hand) that corespond to a given position. If we only care about some of the pieces, then the position including the pieces we don't care about is called a pattern.

Unhanding

A Face algorithm is an algorithm consisting entirely of Face Moves (12a1), and must not include any Outer Block Moves (also known as Wide Moves). A Hand algorithm is an algorithm which may include Outer Block Moves, Middle Layer, Slice, Anti-Slice, Grouping, Repetitions, or Rotations. There are an infinite number of Hand algorithms for each Face algorithm. The process of converting a Hand algorithm into a Face algorithm is called unhanding.

For example, take y2 L U L', which is a hand algorithm. In order to convert it into a face algorithm, we need to get rid of the rotation in such a way that only face moves remain: R U R'.

Parallel move compression

To facilitate making a database, there are several canonicalizations that can be performed, like unhanding, but may also include compressing multiple parallel moves into one or two moves. For example the moves L R L R L can be compressed into L' R2 or R2 L' (but according to the database canonicalization algorithm, it should be the former if parallel moves are to be listed in alphabetical order).

Parallel move collation

Many algorithms involving middle layer moves become two face moves when unhanded. These two moves should always be listed in alphabetical order. For example, R L' becomes L' R.
